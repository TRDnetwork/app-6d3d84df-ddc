This schema implements a backend for an online store with product catalog, discount validation, and order tracking. The `app_24de_products` table stores product information with public read access. `app_24de_discount_codes` manages active discount codes with validation logic. `app_24de_orders` records completed purchases (cart logic remains client-side in localStorage). Two edge functions are provided: `validate-discount` checks discount code validity and returns percentage, and `create-order` processes order confirmations. A public storage bucket `app_24de_product_images` handles product images. Products and orders tables are added to realtime publication for potential admin dashboards. RLS policies ensure proper access control: products are publicly readable, discount codes are publicly viewable when active, and orders are insertable by anyone but viewable only by the customer who placed them.
